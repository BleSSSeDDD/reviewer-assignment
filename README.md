# reviewer-assignment
Backend service for automatic PR reviewer assignment.

Этот сервис предназначен для **автоматического распределения ревьюверов на pull request'ы** внутри команд разработки. Его ключевые функции:

1.  **Управление командами и пользователями**: Позволяет создавать команды, добавлять/удалять участников, а также управлять их активностью.
2.  **Создание Pull Request'ов (PR)**: При создании PR система автоматически назначает до 2 ревьюверов из команды автора.
3.  **Управление состоянием PR**: Возможность помечать PR как завершённые (`MERGED`) после слияния.
4.  **Переназначение ревьюверов**: Если ревьювер недоступен, его можно заменить на другого активного участника из той же команды.
5.  **Просмотр назначений**: Пользователи могут посмотреть список всех PR, в которых они назначены на ревью.

Сервис обеспечивает логическое распределение задач код-ревью внутри команд, следя за тем, чтобы ревьюверами становились только активные коллеги.

В качестве хранилища используется postgresql, ручки сгенерированы при помощи генератора openapi, тестирование проводилось с использованием grafana/k6

# Запуск:

```bash
docker compose up
```

Образ сервера запушен на докерхаб, поэтому для запуска нет необходимости скачивать исходный код.

Сервер был протестирован, подробные результаты тестов можно посмотреть в файле test-results.md, короткий вывод:

Сервер стабильно обрабатывает до 500 RPS с задержкой менее 2 секунд и минимальными ошибками (<0.3%), при пиковой нагрузке в 1000 пользователей сохраняется работоспособность с увеличением задержки до 6 секунд.
